지금 당장 할 일 (순서대로)
1. 모몽가에 로그인해서, 설문(앙케트)을 하나 직접 만들어보세요.
질문 4개만 넣으세요: 이름 / 메일 / 전화번호 / 문의내용. (슬라이드 3번 왼쪽 화면이 바로 이 작업입니다)

2. 그 설문의 응답 링크로 직접 들어가서, 본인이 답변을 제출해보세요.
남이 아니라 본인이 방문자인 척 답변 넣고 "전송" 눌러보는 거예요.

3. 메일이 실제로 오는지 확인하세요.
접수완료 메일(본인에게), 통지 메일(담당자 계정에게) 두 개 다 오는지.

4. 관리 화면에 들어가서, 방금 낸 답변이 보이는지 확인하세요.
검색해서 나오는지, CSV 다운로드 버튼이 있는지도 같이 확인.

5. 여기까지 하고 나서, 소스코드를 열어서 "3번 답변이 제출됐을 때 실행되는 코드"를 찾아보세요.

어느 Controller가 이 요청을 받는지
어느 Service가 DB에 저장하는지
메일 보내는 코드가 어디 있는지
답변 데이터가 DB에 어떤 모양으로 저장되는지 (테이블 열어서 직접 확인)
여기까지가 이번 주 목표입니다
이 5단계가 끝나면 "모몽가가 원래 뭘 할 수 있는지"를 완전히 파악한 겁니다. 그 다음(폼 코드 자동생성, 별도 API 만들기, 보안 등)은 전부 이걸 다 해본 다음에 설계 단계에서 결정할 일이라, 지금 머릿속에 담아둘 필요 없어요.

즉, 오늘 할 일 = "모몽가로 설문 하나 만들어서 직접 써보고, 그때 코드가 어떻게 도는지 눈으로 추적하기" 딱 이거 하나입니다.
