TOPIK 회독 단어앱 - 21일 상세 개발 일정표 (Cursor 지시문 포함 최종본)

이 문서는 지금까지 정리한 기능/규칙/개발 방향을 기준으로,
개발 처음 하는 사람도 그대로 따라갈 수 있게 만든
"넉넉한 일정표 + 해야 할 일 + Cursor에게 실제로 넣을 지시문" 문서입니다.

기준:
- 하루 3~5시간 작업 기준
- Cursor + Expo + React Native + SQLite 구조
- MVP 우선
- 통계 기능 제외
- Free / Premium / 14일 trial / notebook 그룹 기능 포함

==================================================
[전체 목표]
==================================================
최종적으로 아래가 동작하는 앱을 만드는 것을 목표로 한다.

1. 첫 화면
- TOPIK 1급 ~ 6급
- 나만의 단어장

2. 학습 구조
- Step 회독
- 알고있음 / 모름 / 정답 보기
- 모름 단어 반복

3. Free / Premium
- Free: TOPIK 1~3급, 뜻만
- Premium: TOPIK 1~6급, 뜻+예문+음성, notebook import/export
- 설치 후 14일 trial
- one-time purchase

4. notebook
- 별표 저장
- 직접 추가
- 그룹(사용자 정의 Step) 생성
- 그룹별 학습
- Excel import/export

5. UI/UX
- 다크/라이트 모드
- 폰에서 바로 확인 가능한 Expo 구조

==================================================
[작업 원칙]
==================================================
1. 한 번에 다 만들려고 하지 않는다.
2. 하루 목표를 작게 잡고 끝낸다.
3. 화면이 뜨는지 먼저 확인한다.
4. 로직은 그 다음 붙인다.
5. 디테일은 만들면서 수정한다.
6. Free / Premium 규칙은 계속 확인한다.
7. 통계 기능은 넣지 않는다.
8. 중요한 코드는 한국어 주석으로 적는다.
9. 매일 Cursor가 뭘 만들었는지 요약받는다.
10. 안 맞으면 그날 바로 수정시킨다.
