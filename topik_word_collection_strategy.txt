# TOPIK 단어 데이터 수집/정리 방식 (최종 정리)

이 문서는 내가 하려는 **단어 데이터 수집 방식**을 정리한 실행용 문서입니다.
목표는 **저작권 문제를 피하면서**, TOPIK 1급 같은 급수별 필수 단어를 앱용 데이터로 정리하는 것입니다.

---------------------------------
1. 기본 원칙
---------------------------------
- 단어 자체는 여러 자료를 참고해서 모을 수 있다.
- 하지만 사이트의 설명, 번역, 예문을 그대로 복사해서 앱에 넣으면 위험할 수 있다.
- 따라서 **단어만 참고/수집**하고,
  **뜻 / 예문 / 음성용 텍스트는 새로 만든다.**

즉:
- 단어 목록 참고 = 가능
- 뜻/예문/번역 복붙 = 금지

---------------------------------
2. 내가 할 방식
---------------------------------
1) TOPIK 관련 자료를 여러 개 본다.
   - 정부 자료
   - 공개 학습 자료
   - 사설 TOPIK 단어 자료

2) 각 자료에서 "단어"만 모은다.
   - 예: 가다, 오다, 학교, 친구 ...

3) 여러 자료에서 겹치는 단어를 정리한다.
   - 중복 단어 제거
   - 표기 차이 정리

4) AI에게 필터링을 맡긴다.
   예:
   "TOPIK 1급 수준 단어 중에서 여러 자료에 자주 겹치고 반드시 외워야 하는 단어만 800개 추려줘"

5) 추려진 단어를 최종 리스트로 만든다.

6) 각 단어의 일본어 뜻은 새로 만든다.
   - 직접 작성
   - 또는 AI로 새로 생성

7) 각 단어의 예문도 새로 만든다.
   - 직접 작성
   - 또는 AI로 새로 생성

8) 최종적으로 CSV/DB용 데이터로 변환한다.

---------------------------------
3. 절대 하면 안 되는 것
---------------------------------
- 특정 사이트의 단어 설명을 그대로 복사하기
- 특정 사이트의 일본어 번역을 그대로 복사하기
- 특정 사이트의 예문을 그대로 복사하기
- 사이트 전체를 그대로 크롤링해서 앱 DB에 넣기
- 사설 자료를 거의 원본 그대로 재배포하는 형태로 쓰기

---------------------------------
4. 안전한 방식
---------------------------------
안전한 방식은 아래와 같다.

- 여러 자료 참고
- 단어만 수집
- AI로 중복 제거 + 중요도 필터링
- 뜻 새로 생성
- 예문 새로 생성
- 앱용 DB로 재구성

즉, 핵심은:
"원본 자료를 앱에 그대로 복사하는 게 아니라, 참고해서 새 데이터셋을 만드는 것"

---------------------------------
5. TOPIK 1급 예시 방식
---------------------------------
예를 들어 TOPIK 1급을 만들 때:

1) 정부 자료 + 공개 학습 자료 + 사설 자료를 여러 개 본다.
2) 1급 단어 후보를 전부 모은다.
3) AI에게 이렇게 요청한다:

"TOPIK 1급 수준 단어 중에서 여러 자료에 공통으로 자주 등장하고,
시험 준비에 꼭 필요한 단어만 800개 추려줘.
중복 제거해서 최종 리스트 형태로 정리해줘."

4) 그 결과를 기준으로 최종 800개 리스트를 만든다.
5) 각 단어에 대해 일본어 뜻과 예문을 새로 생성한다.

---------------------------------
6. 뜻 / 예문 생성 방식
---------------------------------
예:
단어 = 가다

AI 요청 예시:
- 이 단어의 일본어 뜻을 초급 학습자용으로 간단하게 만들어줘.
- 이 단어의 한국어 예문 1개, 일본어 예문 1개를 새로 만들어줘.

중요:
- "사이트 문장 복사"가 아니라
- "AI 새 생성"으로 간다.

---------------------------------
7. 최종 데이터 구조 예시
---------------------------------
CSV 예시 컬럼:
- level
- word_id
- word_order
- korean
- japanese
- pronunciation
- meaning_short
- example_ko
- example_ja
- part_of_speech
- tags
- difficulty

즉, 최종 앱에는
"정리된 단어 리스트 + 새로 만든 뜻 + 새로 만든 예문"만 들어간다.

---------------------------------
8. 이 방식의 장점
---------------------------------
- 저작권 위험을 줄일 수 있다.
- 여러 자료를 종합해서 더 좋은 리스트를 만들 수 있다.
- 앱에 맞는 구조로 재구성 가능하다.
- free/premium용 데이터 분리도 쉬워진다.

---------------------------------
9. 한 줄 핵심 정리
---------------------------------
내가 할 방식은:
"TOPIK 관련 여러 자료에서 단어만 참고/수집하고,
AI로 겹치고 중요한 단어를 추린 뒤,
뜻과 예문은 새로 생성해서 앱용 데이터셋으로 만드는 방식"이다.
